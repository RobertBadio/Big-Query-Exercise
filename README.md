📊 Retail Sales Analysis – BigQuery Project
📌 Overview

This project contains SQL solutions for a practical data analytics exercise using Google BigQuery. The dataset focuses on retail sales transactions, and the queries are designed to extract insights, perform transformations, and apply analytical logic.

The exercise covers core SQL concepts including filtering, aggregation, grouping, and conditional logic.

📂 Dataset
Name: Retail Sales Dataset
Platform: Google BigQuery
Contents: Customer transactions, product categories, pricing, quantities, and demographic data
🎯 Objectives

The project answers key business questions such as:

Filtering transactions by time period
Identifying high-value transactions
Calculating total revenue
Segmenting customers by age
Analyzing product performance
Classifying spending behavior
🧠 SQL Concepts Covered
WHERE clause (filtering data)
Aggregate functions (SUM, AVG)
DISTINCT
GROUP BY
HAVING
CASE statements (data classification)
Conditional aggregation
Calculated fields
📊 Key Tasks Implemented

Based on the exercise:

✔️ Data Filtering
Transactions from a specific year (2023)
✔️ Advanced Conditions
Transactions above average total amount
✔️ Aggregation
Total revenue calculation
✔️ Data Exploration
Unique product categories
✔️ Group Analysis
Quantity sold per product category
✔️ Customer Segmentation
Age groups:
Youth (<30)
Adult (30–59)
Senior (60+)
✔️ Transaction Classification
High-value transactions (Total Amount > 500)
✔️ Revenue Filtering
Categories with revenue above 5000
✔️ Pricing Classification
Cheap, Moderate, Expensive (based on unit price)
✔️ Spending Behavior
High vs Low spenders for customers aged 40+
